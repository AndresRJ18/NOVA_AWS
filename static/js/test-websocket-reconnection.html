WebSocket Reconnection Logic Test

This test validates that the WebSocket client implements reconnection logic according to requirements 8.3 and 8.4:

  • ✓ Detect WebSocket disconnection events
  • ✓ Attempt automatic reconnection up to 3 times
  • ✓ Use 1 second delay between reconnection attempts
  • ✓ Activate text fallback if all reconnection attempts fail
  • ✓ Notify user of reconnection status

Test 1: Reconnection Attempt Count

Verifies that the client attempts exactly 3 reconnections before giving up.

Test 2: Reconnection Delay

Verifies that the delay between reconnection attempts is 1 second (fixed, not exponential).

Test 3: Fallback Activation

Verifies that after 3 failed reconnection attempts, an error with code 'websocket_reconnect_failed' is triggered.

Test 4: Reconnection Notifications

Verifies that the onReconnecting callback is called with correct attempt numbers.
